最初に結論

これまでChatGPTとCodexを組み合わせて使ってきましたが、今回からメインのコーディング環境をClaude Codeへ乗り換えることにしました。理由は、自然言語での指示だけで作業を進められる点と、コーディングに関してはChatGPTの上位互換だと感じたことです。ただし、乗り換えたばかりで具体的な成果はまだありません。この記事はその出発点の記録です。

背景

これまでの自分の使い分けは、ChatGPTで考えを整理し、Codexで実際のファイルを修正するという流れでした。この方法自体は初心者でも進められましたが、Codexとのやり取りでは指示の出し方や確認作業に手間を感じる場面がありました。Claude Codeという選択肢を知り、実際の作業をどこまで任せられるのか気になったのが、乗り換えを検討したきっかけです。

実際に行ったこと

まずはClaude Codeを実際に触ってみるところから始めました。ターミナル上で自然な日本語のまま「これを直したい」「この機能を追加したい」と伝えると、そのまま作業が進んでいく感覚があり、これまでのツールより指示のハードルが低いと感じました。コーディングに関する理解や修正の的確さも、ChatGPTでの作業と比べて上位互換だと感じる場面がありました。

今回は、このサイト自体の記事追加や更新作業を、Claude Codeを使って進める最初のケースとして扱っています。まだ触り始めた段階なので、大きな成果を語れる状況ではありません。

つまずいた点

正直なところ、乗り換え初日の時点でははっきりした「失敗」も「大きな成功」もありません。強いて言えば、ChatGPTやCodexで慣れていた指示の出し方をそのまま使ってよいのか、Claude Code独自の勝手がどこにあるのか、まだ手探りの部分があります。この記事自体、結論を急がず現状をそのまま記録することを優先しています。

現時点で分かったこと

今の時点で言えるのは、自然言語で指示できる範囲が広く、コーディング面でChatGPTより一段進んだ印象を受けたということです。ただし、これは使い始めてすぐの感想であり、継続して使った結果どうなるかはまだ分かりません。今後、実際の作業効率や仕上がりの質を、この副業プロジェクトの中で検証していきます。

初心者への結論

ツールを乗り換える時、最初から明確な成果を求めすぎると始めにくくなります。今回のように「魅力を感じたから、まず使ってみる」という段階があってよいと考えています。このサイトでは、Claude Codeを使い続けた結果を、良かった点もつまずいた点も含めて今後の記事で更新していきます。
